# Post-PR #367 Cleanup Audit

**Scope:** `15a3838d` (2026-07-07, "add native-module architecture guard") → `HEAD` (`3cf2ae2`, main)
**Authoritative fix:** PR #367, merge commit `8748fe2` (merged 2026-07-13T20:49:13Z), branch `fix/windows-native-oom-phone-mirror` → `main`
**Root cause (confirmed):** `src/lib/onboarding/orchestrator.ts` ran a perpetual `requestAnimationFrame` drain loop that kept the Chromium compositor permanently non-idle. Under **software compositing** (no real GPU compositor — reproduces on Windows boxes without hardware compositing, and would reproduce on any software-compositing macOS fallback), this caused an unbounded PartitionAlloc raster-tile leak: native RSS climbed lock-step with wall-clock time on a flat JS heap until the process went unresponsive. GPU-composited machines (dev Mac, Metal) never accumulated the leak, which is why it was invisible in local dev. Fixed by `535503d`/`9ac2827` (self-terminating `setTimeout`-based drain, event-gated) and finalized by `c9f9ac6` (one-shot deadline scheduler, revision-cached `getSnapshot()`); a genuine post-merge regression in the new scheduler was caught and fixed the next day by `7477b1aa` (2026-07-14).
**Six independent reviews completed:** git-history/PR#367 deep-dive, Electron startup/packaging, Windows-specific residue, macOS-specific residue, logging/telemetry/security, test-coverage/build verification.

Classification key: **A** = permanent fix, **B** = legitimate production improvement, **C** = temporary diagnostic (evaluate retain-vs-remove on its own merits), **D** = temporary behavior change tied to a since-falsified/closed hypothesis, **E** = test/tooling support, **F** = unrelated feature work (do not touch), **G** = uncertain (resolved below with fresh evidence, not left open).
